Router Retriever cho RAG của bot bán nhà

Nghe cả câu chuyện của khách → truy hồi đúng đoạn dữ liệu → trả lời & bật slide. Bản thiết kế nâng tầng truy hồi từ hybrid đơn giản lên Router Retriever (BM25 + Vector + RRF + Reranker), kèm số liệu baseline đo được.

01

High Level — luồng RAG hiện tại

Câu hỏi của khách đi qua truy hồi → rerank → nhồi vào prompt → LLM trả lời chỉ dựa trên ngữ cảnh.

Documents → Vector Database
↓
User Question → Retriever Pipeline → Retrieve Chunk → Rerank + Refine
↓
Prompt: “Given {context}, answer {question} — chỉ dựa trên ngữ cảnh, không có thì nói không biết.” → LLM → Answer
02

Router Retriever — bản nâng cấp truy hồi

04

Điều số liệu hé lộ

Vì sao cần tầng vector + reranker chứ không chỉ BM25.

Câu “Chủ đầu tư Nhã Đạt là ai?” chỉ xếp hạng #6

BM25 bỏ dấu nên nhã đạt → nha dat, trùng với nhà đất (real estate) xuất hiện khắp corpus → kéo nhầm đoạn chung chung lên đầu. Đây đúng là loại lỗi mà semantic vector + reranker của Router Retriever xử lý được — bằng chứng cụ thể để ưu tiên dựng hai tầng đó trước.
